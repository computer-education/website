## Mengobrol dengan Komputer via Teks

Kita terbiasa menggunakan **GUI (Graphical User Interface)**—mengoperasikan komputer menggunakan Mouse dengan cara klik ikon, geser _window_, dan tarik-lepas (_drag & drop_).

Tapi sebelum GUI ditemukan, cara satu-satunya memerintah komputer adalah dengan mengetikkan barisan instruksi teks. Layar hitam tempat kita mengetik teks inilah yang disebut **Command Line Interface (CLI)** atau sering disebut **Terminal / Command Prompt**.

### Mengapa CLI masih digunakan hari ini?
Mungkin kamu bertanya: _"Buat apa capek-capek ngetik kalau bisa di-klik?"_. Jawabannya: **Kecepatan dan Otomatisasi**.

1. **Jauh lebih cepat:** Bagi _programmer_ dan _SysAdmin_, mengetik satu baris kode di Terminal jauh lebih cepat daripada harus mengklik 10 menu yang berbeda di tampilan grafis.
2. **Kendalikan Komputer Jarak Jauh:** Server-server raksasa di pusat data Google atau Amazon tidak memiliki layar monitor (Apalagi Mouse!). Cara satu-satunya mengendalikan komputer server tersebut adalah dengan mengirimkan perintah teks CLI jarak jauh.

### Contoh Perintah CLI Sederhana (Linux/Mac)
- `pwd` (Print Working Directory): Mengecek posisi kita sedang berada di folder mana.
- `ls` (List): Menampilkan daftar file dan folder yang ada di lokasi kita saat ini.
- `cd Desktop` (Change Directory): Pindah masuk ke dalam folder Desktop.
- `mkdir Tugas` (Make Directory): Membuat folder baru bernama "Tugas".

Jika kamu bercita-cita menjadi Programmer, memahami dasar-dasar CLI / Terminal adalah kemampuan yang **wajib dikuasai**.
